Lesson: Object Oriented Programming

Summery
Object Oriented Programming (OOP) is a programming paradigm for a style of programming in which classes are based on the concept of "objects". Each object has an identity, state (properties that may change), and a behavior (it can do different things). Software objects have all of these as well.

Classes
Classes are descriptions for an object, either predefined java library class or defined using java.

Sub classes
Sub classes are classes that extend a superclass. They inherit and therefore can use all the public methods of a super class. In it's constructor, a subclass must use the super class constructor in the first line of it's constructor or rely on the default no argument constructor. It will also usually have it's own fields (attributes) which define what makes it its own object. For example, a shape will have numberOfSides, and area as variables, where as a subclass of the shape class, say the circle, may additionally have diameter and radius variables.

Fields (Attributes)
Objects in real life, have attributes. These can be colors, size, name, and much more. In object oriented programming, these attributes are represented as variables, or fields, in an objects class. These should always be private and not accessible directly by subclasses. Instead, they can be accessed by set and get methods.

Abstract Classes
Abstract classes are classes with at least one abstract method. Abstract methods are methods with only signatures defined and that is declared abstract. Abstract classes can be used as super classes for specific classes and describe abstract objects. They can also have constructors that can be called by sub classes constructors. Java does not actually allow objects to be created of abstract classes.

Interface An interface is very similar to a class, however all of it;s methods are abstract, and cannot have fields or constructors. When a class implements an interface, it must have all the methods outlined in the interface. One class can implement more then one interface. Objects of a class that implements and interface are supplied wit a secondary data type. Like an abstract class, interfaces cannot be instantiated.
